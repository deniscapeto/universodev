O que é esta lista?

Esta simples lista serve apenas para guardar o que estou apredendo com a minha experiência diária. O aprendizado de cada desafio novo costuma virar um novo tópico aqui.

Na hora do desenvolvimento, não se esqueça:

  • Documente sua API (docstrings, swagger e etc);
  • Remova código comentado antes do commit;
  • Não use Magic Numbers;
  • Crie testes unitários:
    • Teste todas as condições (branchs);
    • Arrange -> Act -> Assert
  • Faça testes de integração, veja o código rodar como seu cliente verá;

Troubleshooting (caçando bugs):

  • Problemas devem ser encarados com frieza, não entre em pânico!
  • Confira valores por inteiro, não compare por partes:
    • Ex: 1923293 é diferente de 1926293
  • Confira as variáveis de configuração (ambiente, arquivos de configuração e etc)
